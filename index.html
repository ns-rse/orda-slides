Automate Uploading from GitHub

GitHub Workflow release_to_ORDA well documented README.md (thanks Bob 🙏)

  1. Create an Article on ORDA
  2. Generate a FIGSHARE_TOKEN
  3. Store FIGSHARE_TOKEN token in repository
  4. Find FIGSHARE_ARTICLE_ID
  1. Copy ORDA.yaml to repository, inserting FIGSHARE_ARTICLE_ID
  2. Create a badge and add to README.md
  3. Commit & push to GitHub
  4. Create a release on GitHub

Fortunately Bob Turner created the releas_to_ORDA repository which includes a GitHub Workflow file ORDA.yaml which you can drop into your .github/workflows/ORDA.yaml and with a little configuration have release artefacts uploaded from GitHub to ORDA automatically.

Its well documented with a comprehensive README.md that tells you all the steps you need to take.

How to get a Personal Token

  • Login to ORDA
  • User Menu > Applications
  • Create Personal Token
  • Description
  • Save

Using secrets in GitHub Actions

  • Repository Settings > Secrets and Variables > Actions
  • Secrets Tab click New repository secret
  • Give it a name e.g. ORDA
  • Paste Token from ORDA in.
  • Add Secret

Login to ORDA and from the User Menu select Applications, scroll down and you can Create Personal Token, add a Description and click on Save. As with most tokens these are only shown once so make sure to copy and save it somewhere secure if you’re not undertaking the next step immediately.

Copy & Edit ORDA.yaml

name: Release to ORDA
on:
  workflow_dispatch:
  release:
    types: [published]
jobs:
  upload:
    runs-on: ubuntu-latest
    env:
      ARCHIVE_NAME: ${{ github.event.repository.name }}-${{ github.event.release.tag_name }}
    steps:
      - name: prepare-data-folder
        run : mkdir 'data'
      - name: download-archive
        run: |
          curl -sL "${{ github.event.release.zipball_url }}" > "$ARCHIVE_NAME".zip
          curl -sL "${{ github.event.release.tarball_url }}" > "$ARCHIVE_NAME".tar.gz
      - name: move-archive
        run: |
          mv "$ARCHIVE_NAME".zip data/
          mv "$ARCHIVE_NAME".tar.gz data/
      - name: upload-to-figshare
        uses: figshare/github-upload-action@v1.1
        with:
          FIGSHARE_TOKEN: ${{ secrets.FIGSHARE_TOKEN }}
          FIGSHARE_ENDPOINT: 'https://api.figshare.com/v2'
          FIGSHARE_ARTICLE_ID: 123456789  # Change this number!
          DATA_DIR: 'data'

This is easy, make a copy of the ORDA.yaml from the repository and save it in your repositories .github/workflows/ORDA.yaml. Edit the value of FIGSHARE_ARTICLE_ID

Add badge to README.md

[![ORDA](https://img.shields.io/badge/ORDA--DOI-10.123456%2Fshef.data.123456789.v.1-lightgrey)](https://figshare.shef.ac.uk/articles/software/TopoStats/123456789/1)

You can add a badge to you repositories README.md which makes it easier for people to find and cite your work, one of the key ideas behind the FAIR principles. The URL for the badge will depend on your DOI and the link will depend on the URL of your package on ORDA.

Once done stage and commit all changes, this will be the README.md you’ve just modified and adding the .github/workflows/ORDA.yaml. Push these to GitHub.

Make a Release

  • Choose Tag (create if required).
  • Add a title.
  • Generate release notes (edit if required).
  • Set as a pre-release if required.
  • Publish release

The process of making a release is unchanged, go to the repository and then navigate to releases/new, choose a tag and create if required. Add a title and you can get a summary of changes by auto-generating the release notes which pulls in the title of commits (only really useful if you’ve clear and effective PR headings). Set as a pre-release if required and then Publish.
